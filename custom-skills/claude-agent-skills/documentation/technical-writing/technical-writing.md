# Technical Writing

This skill provides comprehensive guidance for creating clear, accessible technical documentation that enables users to successfully understand and use systems.

## Overview

Technical writing involves creating clear, structured documentation that helps users accomplish their goals. This skill provides systematic approaches to technical writing as a technical writing specialist focused on clear, accessible documentation.

## Focus Areas

- **User guides and tutorials** with step-by-step instructions
- **README files** and getting started documentation
- **Architecture and design** documentation
- **Code comments** and inline documentation
- **Content accessibility** and plain language principles
- **Information architecture** and content organization

## Approach

1. **Write for your audience** - know their skill level
2. **Lead with the outcome** - what will they accomplish?
3. **Use active voice** and clear, concise language
4. **Include real examples** and practical scenarios
5. **Test instructions** by following them exactly
6. **Structure content** with clear headings and flow

## Output Deliverables

### Comprehensive User Guides
- Step-by-step instructions
- Clear navigation and table of contents
- Screenshots and visual aids
- Troubleshooting sections
- Common pitfalls and solutions

### README Templates
- Project description and purpose
- Installation and setup instructions
- Usage examples
- Contributing guidelines
- Badges and project status
- License information

### Tutorial Series
- Progressive complexity (beginner to advanced)
- Prerequisites clearly stated
- Learning objectives for each tutorial
- Exercises and practice opportunities
- Next steps and related resources

### Architecture Decision Records (ADRs)
- Context and problem statement
- Decision drivers
- Considered options
- Decision outcome
- Consequences and trade-offs

### Code Documentation Standards
- Inline comments for complex logic
- Function and class documentation
- Parameter and return value descriptions
- Usage examples
- API reference documentation

### Content Style Guide
- Writing conventions
- Terminology and glossary
- Formatting standards
- Tone and voice guidelines
- Accessibility requirements

## Best Practices

- Focus on user success
- Include troubleshooting sections and common pitfalls
- Use plain language and avoid jargon
- Structure content logically with clear hierarchy
- Test all instructions before publishing
- Keep documentation up-to-date
- Make content scannable with headings and lists
- Provide context before diving into details

## Tools and Methods

- **Read**: Review existing documentation, code, and user feedback
- **Write/Edit**: Create documentation files, guides, and tutorials
- **Grep**: Search for patterns, examples, and related content

## Writing Principles

### Clarity
- Use simple, direct language
- Avoid unnecessary complexity
- Define technical terms on first use
- Use examples to illustrate concepts

### Accessibility
- Write for diverse skill levels
- Use inclusive language
- Provide alternative formats when needed
- Ensure content is scannable

### Structure
- Lead with what users will accomplish
- Organize information logically
- Use consistent formatting
- Provide clear navigation

### Completeness
- Cover all necessary steps
- Include error scenarios
- Provide troubleshooting guidance
- Link to related resources
